# 场景设计文档

这个目录放的是**按具体场景划分**的设计文档。

每个场景一个子目录，避免不同假设、不同传感器、不同飞控约束混在一起。

## 设计目的

顶层设计负责讲“共性”，场景目录负责讲“特化”。

例如不同场景可能会有明显差异：

- 室内 vs 室外
- 无 GPS vs 有 GPS
- 激光雷达 vs 视觉 vs 多传感器融合
- ArduPilot vs 其他飞控

这些差异如果都堆在同一份总文档里，后续会越来越混乱，所以按场景拆目录更清晰。

## 当前场景

当前最明确、正在推进的是：

- `indoor/`

后续如果需要，可以继续新增：

- `outdoor/`
- `hybrid/`
- `warehouse/`
- 其他任务或环境特化场景

## 每个场景目录建议包含的内容

- `README.md`: 场景目标、约束、入口
- `architecture.md`: 该场景下的系统拆分和数据流
- `mvp_plan.md`: 该场景的阶段目标和验收标准
- 其他补充设计文档：例如定位、飞控桥、传感器方案、任务流程

## 与通用文档的关系

- 先在 `docs/general/` 定义共性原则
- 再在 `docs/scenarios/<scene>/` 写该场景的特化实现
- 如果某条内容只对一个场景成立，应优先写到场景目录，而不是回写到通用文档
