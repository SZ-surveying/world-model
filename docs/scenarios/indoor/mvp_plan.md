# 室内场景 MVP 路线图

## 1. MVP 定义

当前室内场景的第一期 MVP 不是“学会所有任务”，而是满足以下条件：

- 室内无 GPS 条件下可以建立稳定定位闭环
- 可以把定位和局部环境状态送入规划链路
- 可以完成基础避障和简单导航
- 输出始终经过安全层后再发送给 ArduPilot
- 在定位异常、传感器异常、模型异常、桥接异常时能稳定降级

## 2. MVP 核心思路

这期 MVP 的重点不在“复杂模型能力”，而在“把闭环打通”。

建议按下面顺序推进：

1. 先打通室内定位闭环
2. 再建立局部地图和简单避障
3. 再引入世界模型做风险预测或评分

## 3. 分阶段路线

### 阶段 0：打通室内定位闭环

目标：

- 跑通激光雷达 / IMU 与 SLAM
- 持续输出位姿和速度
- 把外部导航估计稳定送入 ArduPilot

需要完成：

- 传感器接入
- 时间同步
- 外参管理
- SLAM 输出检查
- ExternalNav 桥接

验收标准：

- 外部导航输入持续稳定
- 室内低速悬停或平移可控
- 坐标系转换正确
- 飞控状态不过度报错

### 阶段 1：建立局部地图和基础避障

目标：

- 从观测构建局部地图和动态障碍表示
- 用规则方法实现最小可用避障

需要完成：

- 点云预处理
- 局部占据图或自由空间表示
- 动态障碍跟踪
- 简单局部规划器
- 安全超时保护

验收标准：

- 能绕开静态障碍
- 面对简单动态障碍能减速、等待或绕行
- 数据链路可录制、可回放

### 阶段 2：引入结构化世界模型

目标：

- 世界模型输入过去若干帧局部状态
- 输出风险图、未来占据或候选轨迹评分

需要完成：

- 统一世界模型输入输出契约
- 在线推理服务
- 回放评估链路
- 与 Planner 的对接

验收标准：

- 输出可被 Planner 稳定消费
- 对动态场景的决策有正向帮助
- 模型失效时可回退到规则方法

### 阶段 3：系统级鲁棒性验证

目标：

- 证明系统在关键故障模式下可降级、可恢复、可诊断

需要完成：

- 故障注入
- 延迟压测
- 回放测试
- 闭环仿真测试
- 人工接管验证

验收标准：

- 定位异常不会直接输出危险命令
- 传感器中断时可触发保守动作
- 通信超时有明确保护路径

## 4. 当前最低可用功能集

如果只保留第一批最小能力，建议包含：

- 室内 SLAM 定位
- ExternalNav 到 ArduPilot 的桥接
- 自机状态统一发布
- 局部地图生成
- 简单动态目标跟踪
- 基础局部避障规划
- 安全裁决与超时保护

在这批能力稳定前，不建议过早追求复杂学习模型上线。

## 5. 当前阶段的开发重点

如果按工程价值排序，当前优先级建议是：

1. 定位闭环稳定性
2. 坐标系与时间同步正确性
3. ArduPilot 桥接稳定性
4. 局部地图和简单避障
5. 世界模型输入输出契约
6. 学习型预测模型

## 6. 当前阶段结论

当前室内场景的 MVP 应该首先证明一件事：

**在室内无 GPS 条件下，系统可以依靠 SLAM 和 External Navigation 建立稳定飞行闭环，并在此基础上逐步接入世界模型。**

只要这条主线成立，后续再增强模型能力才有意义。
